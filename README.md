# 프로그래밍과 데이터 기초 단위 프로젝트

## 프로젝트 주제

- 국내 현대-기아 자동차별(승용, 승합) 신차가격 + 유지 비용(보험비, 유류비, 소모품 등)과 렌트 또는 중고차 10년간의 가격 비교

---

## 프로젝트 목표

- 업무 도메인에 필요한 데이터 정의
- 파이썬 기반 Beautiful Soup과 Selenium활용 데이터 수집
- MySQL DB 설계 및  데이터 저장
- 파이썬과 MySQL 연동 조회 시스템 개발
- Streamlit 활용 데이터 조회 GUI 구현

---

## 프로젝트 내용

- 데이터 조회에 사용할 데이터 정의
- 사용할 데이터 다운로드 및 웹 크롤링 통한 수집
- MySQL 설치 및 환경 설정
- MySQL ERD 설계 및 수집한 데이터 저장
- 파이썬 활용 MySQL 연동하여 SQL로 데이터 조회
- Streamlit 활용 데이터 조회 GUI 구현

---

## 필수 산출물 (Github 업로드)

- 데이터베이스 설계 문서 (ERD)
- 데이터 수집 프로그램 (MySQL)
- 데이터 조회 프로그램 (Streamlit)

---

## 주요 기능

- 자동차 모델별 비용 계산
    - 10년간의 유지 비용 예측(유류비, 보험비, 정비 비용, 소모품 등)
    - 신차, 중고, 렌트, 리스 가격 비교
    - 감가상각 반영(여유 있을 시)
- 모델 간 가격 비교
    - GPU 성능 비교 사이트처럼 두 모델 간 가격 및 유지 비용 비교
    - 각각 조건 다르게 설정 가능하도록
- 가격 예측 로직은 단순하게
    - 여유 있으면 ML, DL 추가
- 여유 있으면 다른 회사 모델도 추가

## 구현 가능성 검토

### 가능한 부분

| 항목 | 방법 | 난이도 |
|---|---|---|
| 현대·기아 신차 가격 수집 | Selenium (JS 렌더링 필요) | 중 |
| 유류비 시세 | 한국석유공사 오피넷 공공 API | 하 |
| 중고차 시세 | 엔카·KB차차차 크롤링 | 중 |
| 소모품·정비비 | 표준 교체 주기 기반 고정값 추정 | 하 |
| 렌트·리스 가격 | 각 렌트카 홈페이지 크롤링 | 중 |
| MySQL 연동 | pymysql / mysql-connector-python | 하 |
| Streamlit GUI | 공식 라이브러리 활용 | 하 |

### 어려운 부분 및 제약

- **보험료**: 개인 조건(나이, 운전경력, 가입 이력)에 따라 편차가 크므로 **특정 조건을 고정**(예: 26세, 최초 가입 기준)하거나 보험개발원 공시 기준 평균값을 사용해야 함
- **현대·기아 공식 홈페이지**: SPA 구조로 Selenium 없이 BeautifulSoup만으로는 가격 데이터 파싱이 어려울 수 있음
- **중고차·렌트 가격**: 실시간 변동 데이터이므로 수집 시점을 고정하고 기준일을 명시해야 함
- **이용약관·robots.txt**: 크롤링 대상 사이트별 허용 여부 사전 확인 필요

## 필요한 데이터

### 수집 대상 데이터 목록

**신차 관련**
- 현대·기아 승용·승합 모델 목록 및 트림별 신차 가격
- 모델별 복합 연비 (휘발유·경유·LPG·전기·하이브리드)
- 연료 종류 및 배기량

**유지 비용 관련**
- 유류비 시세 (오피넷 API — 휘발유·경유·LPG 전국 평균)
- 보험료 기준 데이터 (차량가액 구간별 기준 보험료 또는 평균 보험료)
- 소모품 교체 비용 및 주기

| 소모품 | 교체 주기 | 평균 비용(예시) |
|---|---|---|
| 엔진오일 | 1만 km / 1년 | 6~15만 원 |
| 타이어 | 4~5만 km | 40~100만 원(4개) |
| 브레이크 패드 | 3~4만 km | 10~20만 원 |
| 에어필터 | 2만 km | 2~5만 원 |
| 배터리 | 3~5년 | 10~20만 원 |

**중고차 관련**
- 연식별·주행거리별 중고차 시세 (기준: 동일 모델 출시 후 1~10년)
- 감가상각 곡선 (선택 구현)

**렌트·리스 관련**
- 월 렌트료 (모델별, 약정 기간별)
- 리스 잔존가치 및 월 납입금 기준표
