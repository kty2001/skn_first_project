## 분석 기준

같은 데이터라도 어떤 기준으로 보느냐에 따라 1위 지역이 달라진다는 점을 보여주는 것이 핵심

| 지표 | 계산식 |
|---|---|
| 절대 등록대수 | 해당 연도 시도별 전기차 누적 등록대수 |
| 인구당 보유율 | 전기차 등록대수 ÷ 해당 시도 인구수 |
| 전년 대비 증가율 | (해당 연도 등록대수 − 전년도 등록대수) ÷ 전년도 등록대수 × 100 |

> 유가 변동률과 전기차 증가율의 관계 (예: 호르무즈 해협 봉쇄 → 유가 상승 → 전기차 구매 증가)는 시간 여유가 있을 경우 추가 분석 예정

## 구현 가능성 검토

### 가능한 부분

| 항목 | 방법 | 난이도 |
|---|---|---|
| 시도별 전기차 등록 현황 | 국토교통부 공공데이터포털 API / 자동차 등록통계 파일 | 하 |
| 지역별 인구수 | 통계청 공공데이터 API | 하 |
| 자동차 기업 FAQ 수집 | 각 제조사 홈페이지 크롤링 (BeautifulSoup) | 중 |
| 유류가격 데이터 | 한국석유공사 오피넷 공공 API | 하 |
| MySQL 연동 | pymysql / mysql-connector-python | 하 |
| Streamlit GUI | 공식 라이브러리 활용 | 하 |

### 어려운 부분 및 제약

- **FAQ 대상 기업 선정**: 어떤 자동차 기업의 FAQ를 수집할지 팀 협의 필요
- **지역 단위 확정**: 17개 시/도 전체 vs. 일부 선별 여부 팀 협의 필요
- **시계열 범위**: 최근 3~5개년 데이터 수집 범위 팀 협의 후 확정
- **유가-전기차 상관관계**: 분석 구현 시 시간이 상당히 소요될 수 있어 선택 구현으로 분류
- **FAQ 크롤링 허용 여부**: 크롤링 대상 사이트별 이용약관·robots.txt 사전 확인 필요
